So when building a website as a web developer we have to plan which URLs we wanna support.

So the idea is that we have various URLs, which leads to different results, different responses being sent back to the visitors of our website.

It's about mapping certain actions, certain things that should happen, to certain URLs.

That's the idea behind URLs.

Now what are views?

Well, the URLs are responsible for defining the URLs we wanna support and the actions that should be triggered.

Views then basically are those actions.

Views are all about holding the concrete logic.

The concrete Python code that should be executed for the different URLs.

A View in Django in its simplest form is just a function.

A function full of Python code, which is executed automatically by Django when a request for a certain URL reaches the Django server.

It could also be a class, you have both options when working with the Django.

The code in that views is simply code that handles the incoming request, does something like reach out to a database or whatever and then returns a fitting response.

And in a view you typically perform tasks like loading and preparing data, running any other kind of business logic like encrypting passowrds or something like that.

And then you prepare and return the response data.

For example a HTML document that should be sent back to the client.
