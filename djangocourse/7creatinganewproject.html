A project is then not a collection of totally detached unrelated apps, but instead a Django project is made up of multiple apps, which combined, form the overall project, the overall website.

	    
(DjangoVirtual) [~/.virtualenvs/DjangoVirtual]$ django-admin startproject monthly_challenges                                                                                                   
(DjangoVirtual) [~/.virtualenvs/DjangoVirtual]$ cd monthly_challenges                                                                                                                          
(DjangoVirtual) [~/.virtualenvs/DjangoVirtual/monthly_challenges]$ python manage.py startapp challenges                                                                                        
(DjangoVirtual) [~/.virtualenvs/DjangoVirtual/monthly_challenges]$ ls                                                                                                                          
challenges  manage.py  monthly_challenges
(DjangoVirtual) [~/.virtualenvs/DjangoVirtual/monthly_challenges]$

This is all something you might want to do, to prepare your project for the code we're about to write.
