When we build cloud native applications, we often have one application component talking to another one. Sometimes via some kind of mechanism, like a notification service or a message bus.

And that means that we can have these distributed applications utilizing different services on AWS and they can all communicate to each other as part of one single application that has some kind of workflow or orchestration moving through it.
